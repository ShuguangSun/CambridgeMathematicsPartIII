The proximal gradient method

The proximal gradient method is a method that can be utilized to minimize functions while at the same time regularizing their parameters with the l1-norm. It is part of the Iterative Shrinkage Thresholding Algorithm (ISTA).
ISTA can be used to minimize a function that comprises a sum of two convex functions, one has to be differentiable anywhere, the other one is the l1-norm and therefore not differentiable at 0.
Often, this algorithm is utilized for minimizing a likelihood with respect to an l1-regularization for its parameters.


Before I get to the derivation of the proximal gradient method, I will cover some basics on subgradients/subderivatives.
I hope you will not get annoyed because it will be extremely detailed.

Subderivatives (also called subgradients)

Subderivatives of a function f:I→ℝ at a point θ0 with I being an interval around θ0 are defined as all values of a scalar g that fulfill the following condition:
f(θ)−f(θ0)≥g(θ−θ0)∀θ∈I

The following plot from the wikipedia subderivative page illustrates quite nicely how those subderivatives can look like (red).
Important property that we will need later: If one of the subderivatives of f(θ) around the point θ^ is 0 then θ^ is a local minimum.

More preliminary stuff


Let's assume we would like to minimize a function of the form f(z)=12∥z−θ∥22+λ∥z∥1⏟non-smooth regularizer for z and therefore take the derivative ∂∂zf(z)=(z−θ)+λ∂∂z∥z∥1
Unfortunately the l1-norm is not everywhere (at 0) differentiable so we will use the subderivative and see where it is zero.

The subderivative of f(z) with respect to the j-th component of z is given by 

∂zjf(z)=⎧⎩⎨⎪⎪{zj−θj−λ}[−θj−λ,−θj+λ]{zj−θj+λ}if zj<0if zj=0if zj>0


So when is this subgradient zero?

  1. if θj<−λ, the only way to get the subgradient to zero is via the zj<0 case if zj=θj+λ <0
  2. if θj∈[−λ,+λ]  (see the second case, zj=0) the subgradient is zero at zj=0
  3. if θj>+λ (see the third case, zj=0) the only way to get the subgradient zero is with zj=θj−λ>0 

More formally, that means the minimal value for zj, let's call it z^j is
z^j=⎧⎩⎨⎪⎪θj+λ0θj−λif θj<−λif θj∈[−λ,λ]if θj>λ


This function can be rewritten as:

soft(θj;λ)=sgn(θj)max(|θj|−λ,0)


where

sgn(x)=⎧⎩⎨⎪⎪−101if x<0if x=0if x>0

The soft function is also referred to as soft-thresholding function.

In summary, the minimum of 12∥z−θ∥22+λ∥z∥1 with respect to the j-th component of z is given by soft(θj,λ).

We can also refer to this minimization problem by a different term:
proxR(θ)=argminz(R(z)+12∥z−θ∥22)


with R(z) being a regularizer, in our case the l1 norm.
proxR(θ) is called the proximal operator, minimizing the regularizer R but also at the same time staying proximal (= close) to θ.
And, we just learned, that we can express this proximal operator above as componentwise soft-thresholding:
proxR(θ)=soft(θ,λ)


Finally arriving at the proximal gradient method

After covering some useful basics, let's finally move on to the derivation of the proximal gradient method.
Let's assume we want to iteratively minimize the likelihood function L(θk) (where θk are its parameters after iteration k) with respect to a regularizer function R.
In each iteration, we will quadratically approximate this likelihood using the Taylor approximation and minimize this local approximation. If the likelihood function is convex, this will eventually lead to its minimum.

As a quick reminder: The Taylor approximation/series of a function f at point a is given by

f(x)=f(a)+f′(a)(x−a)+f′′(a)2!(x−a)2+⋯+f(i)(a)k!(x−a)k+hk(x)(x−a)k

with limx→ahk(x)=0 and f(i)(x) being the i-th derivative of f(x).

Since we want to approximate the function only quadratically, we only keep the first the terms of this sum and omit the rest.
We get
q(z)=L(θk)+δTk(z−θk)+γTk12∥z−θk∥22


with δk=∇L(θk) and γk=∇2L(θk).

we will now approximate the second order derivative even more crudely by saying γk=12tkI and will add a regularizer R(z)=λ∥z∥1 and we want to minimize this local approximation by z.
So we get

argminzL(θk)+δTk(z−θk)+12tk∥z−θk∥22+R(z)


The idea is to minimize this approximation to get a new set of parameters θk+1 and repeat this procedure until we have reached the real minimum of our likelihood function.
Since we are minimizing with respect to z, we can just drop all the parts that do not depend on z.

θk+1= argminzδTk(z−θk)+12tk∥z−θk∥22+R(z)


and multiply by tk

θk+1=argminztkδTk(z−θk)+12∥z−θk∥22+tkR(z)=argminztkδkz−tkδkθk+12zTz−zTθk+12θTkθk+tkR(z)

Some people might see now that by adding the constant t2kδTkδk (it is constant so we are allowed to add it cause we are minimizing over z) this sum can be reduced to
argminz12∥z−(θk−tkδk)∥2+tkR(z)

If you can not see that immediately (like me) you can just expand the result and see that really only that constant t2kδTkδk was missing in the equation above.

And now comes the cool stuff where everything starts to make sense.
The part θk−tkδk should look familiar! It is the gradient descent update rule with θk being the parameters, tk the learning rate and δk the derivative of the objective function with respect to the parameters.

So let's try to interpret that formula:
What we essentially do here, is looking for a z that is close to our parameters after the next step of gradient descent and at the same time is regularized towards sparsity by R(z).
AND all this is nicely derived from a second order approximation of the likelihood function. Really amazing.
This method is also called iterative soft-thresholding for our case where R(z)=∥z∥1 since we can solve this by using the soft-thresholding function as θk+1=soft((θk−tkδk),tk).

The final question that remains is what is how to choose tk.
Well, there are several ways to pick it but since we are using 1tkI to approximate the hessian of the likelihood function (∇2L) it makes sense to say
1tk(θk−θk−1)≈δk−δk−1

So we could get a value for 1tk by
1tk=argmin1tk∥1tk(θk−θk−1)−(δk−δk−1)∥22=(θk−θk−1)T(δk−δk−1)(θk−θk−1)T(θk−θk−1)

This stepsize is also known as Barzilai-Borwein stepsize.

Combining the Barzilai-Borwein stepsize with the iterative method derived above and a rule to gradually decrease λ we arrive at an algorithm called SpaRSA or ISTA.

For further reference, I recommend "Machine Learning: a Probabilistic Perspective" by Kevin Murphy. In particular chapter 13.4.3.2.
